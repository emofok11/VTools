# 需求文档

## 引言
本文档描述了自定义描述输入框中列表编辑逻辑的优化需求。目标是参照 Microsoft Word 的项目符号和编号列表的自动化状态机逻辑，实现更智能、更符合用户直觉的富文本编辑体验。这包括对回车键、退格键、Tab键的智能状态转换处理，以及基于特定字符输入的自动套用格式功能。

## 需求

### 需求 1：回车键 (Enter) 的智能处理

**用户故事：** 作为一名用户，我希望在列表项中使用回车键时，系统能根据当前上下文自动判断是继续列表、结束列表还是软换行，以便我能流畅地进行排版。

#### 验收标准
1. WHEN 用户在非空的列表项末尾按下 Enter 键 THEN 系统 SHALL 在下一行自动生成一个同层级、同类型的项目符号或编号，并保持相同的缩进格式。
2. WHEN 用户在一个完全空白的列表项按下 Enter 键 THEN 系统 SHALL 结束当前列表模式，删除该行的项目符号，并将光标所在段落恢复为普通正文样式。
3. WHEN 用户在列表项中按下 Shift + Enter 键 THEN 系统 SHALL 插入一个软换行符，文字另起一行但保持当前列表项的缩进对齐，且不生成新的项目符号。

### 需求 2：退格键 (Backspace) 的智能处理

**用户故事：** 作为一名用户，我希望在列表项中使用退格键时，系统能根据光标位置智能地撤销列表格式、合并段落或提升层级，以便我能方便地修改列表结构。

#### 验收标准
1. IF 当前行是刚自动生成的空列表项 WHEN 用户按下 Backspace 键 THEN 系统 SHALL 撤销该行的列表格式，将其转换为普通的左对齐文本段落。
2. IF 光标位于非空列表项的最前端（项目符号右侧） WHEN 用户按下 Backspace 键 THEN 系统 SHALL 将当前行的内容向上合并到上一个列表项的末尾。
3. IF 光标位于有缩进的列表项（如二级列表）的最前端 WHEN 用户按下 Backspace 键 THEN 系统 SHALL 将该列表项的层级提升一级（例如从二级变为一级），而不是直接删除项目符号。

### 需求 3：自动套用格式 (Auto-Format)

**用户故事：** 作为一名用户，我希望通过输入特定的字符组合能自动触发列表模式，以便我能快速开始编写列表而无需点击工具栏按钮。

#### 验收标准
1. WHEN 用户在段落开头输入 `- ` 或 `* ` 或 `. `（字符加空格） THEN 系统 SHALL 自动将该段落转换为无序列表。
2. WHEN 用户在段落开头输入 `1. `（数字加点加空格） THEN 系统 SHALL 自动将该段落转换为有序列表。
3. WHEN 用户在段落开头输入 `-> `（箭头加空格） THEN 系统 SHALL 自动将该段落转换为箭头样式的列表。
4. IF 系统刚刚触发了自动套用格式 WHEN 用户立即执行撤销操作（如按下 Ctrl + Z 或 Cmd + Z） THEN 系统 SHALL 取消列表属性，并恢复用户输入的原始触发字符。

### 需求 4：Tab 键的层级控制

**用户故事：** 作为一名用户，我希望使用 Tab 和 Shift+Tab 键来快捷调整列表项的层级，以便我能快速组织多级列表。

#### 验收标准
1. IF 光标位于列表项的最前端 WHEN 用户按下 Tab 键 THEN 系统 SHALL 增加当前列表项的缩进层级（降级），并更新项目符号样式。
2. IF 光标位于有缩进的列表项的最前端 WHEN 用户按下 Shift + Tab 键 THEN 系统 SHALL 减少当前列表项的缩进层级（升级），并更新项目符号样式。

### 需求 5：默认状态下的无符号模式

**用户故事：** 作为一名用户，我希望在默认状态下输入文字时前面不会自动出现项目符号，以便我能自由地编写普通文本段落，只有在明确需要列表时才触发列表格式。

#### 验收标准
1. WHEN 用户在输入框中开始输入新内容或新建一行时 THEN 系统 SHALL 默认不添加任何项目符号或编号，保持为普通文本模式。
2. IF 用户没有输入任何触发字符（如 `- `、`1. ` 等） WHEN 用户输入文本内容 THEN 系统 SHALL 保持普通段落格式，不自动转换为列表格式。
3. WHEN 用户在普通段落中输入内容并按回车键 THEN 系统 SHALL 创建一个新的普通段落，而不是自动生成项目符号。
